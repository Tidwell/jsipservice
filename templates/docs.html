Overview

The ipService library provides a single method: .get. It accepts a callback as its' only argument, which is passed an object containing the user's IP address. 

{
  ip: "00.000.000.000"
}

Example

Two, simple steps

1. Include the library via script tag.

<script src="www.jsIpService.com/ipservice.js"></script>

2. call .get.

function callback(user) {
  alert(user.ip);
}
ipService.get(callback);

Internals

Internally, the library caches your callback before injecting a script tag into the page, then on response, executes your callback and performs cleanup (removing the injected script tag) and removing the stored callback.

While the browser is loading the jsonP response from the ipService, the callbacks are exposed via ipService.callbacks. ipService.callbacks also has a default property that will be called if no callback is passed to .get, which simple calls console.log if available. You could overwrite this if you would rather just call .get() instead of passing a named function.

If the warning: 'Overwriting existing ipService variable' shows up in console, either you are including the script tag multiple times, or you are already using the ipService variable in your own code in global scope. If this is the case, feel free to download and rehost a copy of the library with whatever changes you need.
